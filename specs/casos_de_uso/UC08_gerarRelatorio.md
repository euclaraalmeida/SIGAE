| Caso de uso         | UC08 Gerar Relatório  |
| ------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Objetivo            | Permitir a extração de dados estatísticos sobre a ocupação dos espaços e listar agendamentos confirmados para fins de auditoria e planejamento. |
| Requisitos          | **RF020** (Relatórios de uso), **RF021** (Painel de aprovados). |
| Atores              | Administrador (CEF). |
| Condição de entrada | O ator deve estar logado como Admin e devem existir agendamentos registados no sistema |
| Fluxo principal     | 1. O ator acessa o módulo "Relatórios" ou "Dashboard Administrativa". <br> 2. O sistema exibe um painel com opções de relatórios disponíveis (ex: "Taxa de Ocupação por cursos", "Agendamentos por Quadra", "Lista de Aprovados"). <br> 3. O ator seleciona o tipo de relatório desejado (ex: "Ocupação Semanal"). <br> 4. O sistema solicita o período de análise (Data Início e Data Fim). <br> 5. O ator define as datas e clica em "Gerar". <br> 6. O sistema processa os dados e exibe o relatório na tela em formato de gráfico ou tabela, mostrando métricas como: Total de horas reservadas, Espaços mais utilizados, Horários de pico. <br> 7. O ator analisa as informações visuais. |
| Fluxos alternativos | **A1 - Exportar Dados**: O ator deseja guardar os dados para uma reunião. <br> 1. O ator clica no botão "Exportar". <br> 2. O sistema oferece os formatos: PDF (para impressão) ou CSV/Excel (para edição) <br> 3. O ator seleciona o formato. <br> 4. O sistema gera o arquivo e inicia o download. <br> **A2 - Consultar Lista de Aprovados (RF021)** <br> 1. O ator seleciona a opção "Agendamentos Aprovados". <br> 2. O sistema exibe uma lista corrida com todos os eventos confirmados no período, servindo como uma "folha de rosto" para controle de portaria ou segurança. |
| Fluxos de Exceção   |  **E1 - Período sem Dados**: <br> 1. O sistema realiza a busca nas datas informadas, mas não encontra nenhum agendamento. <br>  2. O sistema exibe a mensagem [MSG11] |
