| Caso de uso         | UC09 Realizar LogOut  |
| ------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Objetivo            | Permitir que usuários autenticados (Aluno, DCE, Admin) encerrem sua sessão com segurança, liberando o acesso às funcionalidades restritas do sistema. |
| Requisitos          | **RF001** (Login) *- Logout complementa o fluxo de autenticação* |
| Atores              | Usuário (Aluno, DCE, Admin-CEF) |
| Condição de entrada | O usuário deve estar autenticado no sistema, com sessão ativa. |
| Fluxo principal     | 1. O ator seleciona a opção "Sair" ou "Logout" na interface do sistema. <br> 2. O sistema invalida a sessão do usuário (remove o token/sessão armazenada). <br> 3. O sistema redireciona o ator para a página inicial pública (não autenticada). <br> 4. O sistema não permite mais o acesso às funcionalidades restritas sem novo login. |
| Fluxos alternativos | **A1 – Logout por inatividade**<br> 1. O usuário permanece inativo por um tempo superior ao limite configurado. <br> 2. O sistema invalida automaticamente a sessão por segurança. <br> 3. Na próxima ação, o sistema redireciona o ator para a página de login [UC02], exibindo a mensagem [MSG02]. |
| Fluxos de Exceção   |  **E1 – Falha na invalidação da sessão**: <br> 1. Ocorre falha técnica ao tentar encerrar a sessão (erro no servidor ou armazenamento de sessão). <br>  2.O sistema exibe a mensagem [MSG12] de erro e orienta o usuário a tentar novamente. <br> 3. Caso o erro persista, o sistema recomenda fechar o navegador e contatar o suporte. | 
| Regras de Negócio | **RN1 - Expiração por Inatividade (Timeout)**<br>_Referente ao passo 1 do Fluxo Alternativo A1:_<br>O sistema deve encerrar automaticamente a sessão do usuário após **30 minutos** sem interação.<br><br>**RN2 - Limpeza de Sessão (Segurança)**<br>_Referente ao passo 2 do Fluxo Principal:_<br>Após o logout, o sistema deve garantir que o botão "Voltar" do navegador não permita a visualização de telas cacheadas (cache-control), forçando um novo login para qualquer acesso restrito. |
