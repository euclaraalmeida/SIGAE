| Caso de uso         | UC05 Manter Agendamentos                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
| ------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Objetivo            | Permitir que o aluno logado consulte seu histórico e solicite alterações ou cancelamentos de agendamentos futuros, respeitando os prazos definidos. |
| Requisitos          | **RF010** (Solicitar Cancelamento 24h), **RF018** (Solicitar Alteração 24h), **RF009** (Moderação pelo DCE), **RF011** (Aprovar Alteração). |
| Atores              | Solicitante (Aluno). |
| Condição de entrada | O ator deve estar logado e possuir agendamentos registados |
| Fluxo principal     | 1. O ator acede à opção "Meus Agendamentos" no menu do sistema. <br> 2. O sistema recupera e apresenta uma lista com os agendamentos do aluno, exibindo detalhes como: Data, Horário, Espaço, Status (Ex: "Confirmado", "Pendente de Alteração", "Cancelado").  <br> 3. O ator visualiza os detalhes de um agendamento específico. |
| Fluxos alternativos | **A1 - Solicitar Cancelamento (RF010)**: O ator seleciona a opção "Cancelar Agendamento". <br> 1. O sistema verifica se a antecedência é maior ou igual a 24 horas em relação ao horário agendado [RN1]. <br> 2. O sistema solicita uma justificativa válida para o cancelamento. <br> 3. O ator insere a justificativa e confirma. <br> 4. O sistema altera o status do agendamento para "Cancelamento Pendente" (ou "Em Análise pelo DCE", conforme RF009). <br> 5.O sistema exibe a mensagem [MSG08]. <br> **A2 - Solicitar Alteração (RF018)**: O ator seleciona a opção "Alterar Agendamento" (mudar horário ou local). <br> 1. O sistema verifica a regra de antecedência de 24 horas [RN1]. <br> 2. O sistema permite que o ator escolha um novo horário disponível (reutilizando a interface de Consultar Calendário). <br> 3. O ator seleciona o novo horário e insere a justificativa. <br> 4. O sistema registra a solicitação e define o status como "Alteração Pendente". <br> 5. O sistema notifica que a mudança aguarda aprovação do moderador (DCE).| 
| Fluxos de Exceção   |  **E1 - Prazo Expirado (Menos de 24h)** <br> 1. O ator tenta solicitar cancelamento ou alteração para um horário que ocorrerá em menos de 24 horas. <br> 2. O sistema bloqueia a ação e exibe a mensagem de erro [MSG07]. <br> **E2 - Nenhuma Justificativa Informada** <br> 1. O ator tenta confirmar a solicitação sem escrever a justificativa obrigatória. <br> 2.O sistema apresenta a mensagem de alerta [MSG09]. |
