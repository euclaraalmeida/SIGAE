| Caso de uso         | UC02 Realizar Login                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| ------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Objetivo            | Permitir que usuários (Aluno, DCE, Admin) acessem as funcionalidades restritas do sistema mediante autenticação via SUAP. |
| Requisitos          | **RF001** (Login), **RF002** (Esqueci senha), **RF013** (Integração SUAP) |
| Atores              | Usuário (Ator Genérico) e SUAP (Sistema Externo). |
| Condição de entrada | O usuário deve estar na página inicial e não possuir uma sessão ativa |
| Fluxo principal     | 1. O ator seleciona a opção "Entrar" ou "Login" na barra de navegação do sistema. <br> 2. O sistema exibe o formulário de login solicitando a Matrícula e a Senha do SUAP. <br> 3. O ator preenche os campos com suas credenciais e clica em "Entrar". <br> 4. O sistema envia os dados criptografados para validação no serviço de autenticação do SUAP (API Externa). <br> 5. O SUAP valida as credenciais e retorna a confirmação de sucesso junto com o perfil do usuário (Aluno ou Servidor). <br> 6. O sistema recebe a confirmação, cria uma sessão de acesso segura e identifica o nível de permissão (Aluno, DCE ou Admin). <br> 7. O sistema redireciona o ator para a Home Page logada.|
| Fluxos alternativos | **A1 - Esqueci minha senha**: O ator percebe que não lembra sua senha. <br> 1. O ator clica no link "Esqueci minha senha". <br> 2. O sistema redireciona o ator para a página oficial de recuperação de senha do SUAP (fora do SIGAE). <br> 3. O caso de uso é encerrado. |
| Fluxos de Exceção   |  **E1 - Credenciais Inválidas (Retorno do SUAP)**: O SUAP retorna que a matrícula ou senha estão incorretas. <br> 1. O sistema exibe a mensagem [MSG01]. <br> 2. O sistema limpa o campo de senha e mantém o ator na tela de login. | 
| Regras de Negócio   | **RN1 - Obrigatoriedade**<br>_Referente ao passo 3 do Fluxo Principal:_<br>O botão "Entrar" só deve ser habilitado se ambos os campos (matrícula e senha) estiverem preenchidos. |
